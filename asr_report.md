# 开源语音识别 (ASR) 调研与选型报告

## 1. 方案对比

| 对比维度 | OpenAI Whisper | 阿里 FunASR | Vosk (本次选型) |
| :--- | :--- | :--- | :--- |
| **许可协议** | MIT License | MIT License | Apache 2.0 |
| **语言支持** | 极强 (支持 90+ 种语言互译) | 强 (主打中文及中英混读) | 良好 (支持 20+ 种语言，需下载对应模型) |
| **模型体量与推理速度** | 模型较大 (Base版 ~140MB)，纯 CPU 推理较慢 | 模型中等，CPU 推理速度较快 | **极小** (中文轻量模型仅 ~42MB)，纯 CPU 推理极快 |
| **流式/实时支持** | 原生不支持流式 (需第三方封装) | 支持流式与非流式 | **原生良好支持流式 (Streaming)** |
| **依赖与部署难度** | 依赖 PyTorch 环境，配置较重 | 依赖 PyTorch/ModelScope，较重 | **极简**，仅需 `pip install vosk`，底层纯 C++ 无大型框架依赖 |

## 2. 选型理由

综合考量本地 PC 环境的计算资源现状，本次实验将“占用内存最小”、“极速离线推理”与“部署轻量化”作为核心评判标准。

由于资源环境底层系统开发受限，虽然 Whisper 准确率极高，但其对硬件资源占用过大；相比之下，Vosk 的纯 C++ 底层和极简的 Python 接口（无需 PyTorch 等庞大框架）展现出了极高的工程性价比。其 42MB 的中文轻量级模型即可满足基本的桌面端与嵌入式端语音识别需求，故最终选择 **Vosk** 作为本地实现方案。
